Description

Texts can be messy. It may be hard for computers to process them in their natural state. Computers tend to carry out operations on numbers with more ease. It would be a good idea to convert our text to vectors of numbers. The bag-of-words (BoW) model is a simple yet effective way to extract features from texts.

The first step is to make a vocabulary. A vocabulary includes all unique words from the whole corpus, so the word occurrence frequency plays a rather minor role here. The second step is to create a vector of numbers by matching the occurrence of each word in a body of text to its position in the vocabulary:

sentence_one = "Welcome to natural language processing"
sentence_two = "Bag of words in natural language processing"

vocabulary = ['natural', 'language', 'processing', 'in', 'welcome', 'bag', 'of', 'to', 'words']

sentence_one_vector = [1, 1, 1, 0, 1, 0, 0, 1, 0]
sentence_two_vector = [1, 1, 1, 1, 0, 1, 1, 0, 1]

Objectives

In this stage, your program should:

  1. Randomize the rows in the dataframe;
  2. Split the dataframe into training and test sets in 80:20 proportion;
  3. Create a vocabulary of unique words from the SMS column of the training set;
  4. Create a bag-of-words function that returns a dataframe;
  5. Print the first 200 rows and the first 50 columns of the modified dataframe.

The test set will be converted to a vector of numbers first. So, we advise that you make your bag-of-word model as a function that can be used further. Although the input parameter in this stage is the training set, this function will be used to create a BoW dataframe for the test set in later stages.
